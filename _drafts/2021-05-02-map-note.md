
# hash

hash其实就是某种算法，用key进行计算，生成一个固定长度的值。

hash的一个作用是用来判断数据是否被篡改。

相同key计算hash的值一定是相同的，不同key计算hash有可能是相同的，这个时候就是所谓的hash冲突。

所以map结构存储时，出现hash冲突的时候，需要进行解决。

## 链表

此时把某个hash的值存储结构使用链表进行存储，所以出现重复的时候，就往链表进行添加。

进行get操作的时候，需要遍历链表，对key进行equal比对，找到key对应的value。

## 开放定址

当产生冲突的时候，此时按照某种规则去寻找一个空闲的位置进行存储。

## 再hash

这种方式就是某个hash冲突后，使用另一种hash方法进行计算，直到不冲突。

增加了计算时间。

## 公共溢出区

将哈希表分成基本表和溢出表两部分，hash冲突时统一存放在溢出表。
